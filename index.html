The Full Story

From broken CSV exports to a plugin
that never drops a product.

1
The Challenge: 4,068 products, no clean migration path
AMB Tobaccos & More needed to migrate their entire OpenCart store to WooCommerce. The store had 4,068 products across 221 Greek-language categories, thousands of images, manufacturer brands, special prices and complex hierarchies. The official importer kept failing on hundreds of products.
2
Phase 1: Custom OpenCart CSV Exporter Plugin
Built a custom admin plugin with auto-fixes: duplicate SKU resolution, image URL encoding, double-round HTML entity decoding for Greek text, empty category fallbacks. After stress testing: 0 empty SKUs, 0 duplicates, 0 broken URLs.
products_READY.csv: 4,063 products | 0 issues found
3
Problem: 548 products failing, all products showing as “Private”
Imported 3,513 but 548 failed — all due to missing images on the old server. Separately discovered post_status was mapping to “Private” instead of “Published”, hiding products from customers entirely.
4
The Decision: Abandon CSV. Build a direct API transfer system.
Built a two-plugin system: a REST API on OpenCart exposing all store data, and a WordPress plugin pulling and importing directly. No CSV. No mapping screens. No mismatches. Direct transfer with real-time progress and automatic retry.
5
Critical Bug: Only 3 categories out of 221
The API was hardcoded to language_id = 1 (English). The Greek store used language_id = 2. 218 categories were invisible.
Before: hardcoded language_id = '1' -> 3 categories After: getLangId() auto-detects -> 221 categories
6
Deep Stress Test: 26 bugs found and fixed
Systematically questioned every assumption before shipping. Found 8 bugs in the OpenCart API and 18 in the WordPress plugin. Critical issues included hash_equals fallback, unstable ORDER BY pagination drift, PHP parse errors from Unicode in source code, and category hierarchy never resolving.
7
Result: 4,068 products. 221 categories. 0 failures. 56 minutes.
Transfer ran 19:49 to 20:45. One HTTP 504 timeout at page 67 was automatically retried and recovered. Every Greek category imported with correct hierarchy. Every product published immediately.
[20:45:11] Complete! 4018 imported, 50 updated, 0 failed

PROJECT AGORA

Proposal Bidding
Extension

A custom proposal and bidding system for OpenCart enabling buyers and sellers to negotiate prices within the store. Named after the ancient Greek marketplace.

OpenCart ExtensionCustom WorkflowBuyer-Seller NegotiationAuto Order Creation
📋
Proposal Submission
Buyers submit custom price proposals on any product. Captures quantity, proposed price, notes, and contact details.
💬
Counter-Offer System
Sellers accept, reject, or counter any proposal. Full conversation thread per proposal with timestamps and status tracking.
🔔
Email Notifications
Both parties notified at every step. Customizable templates using OpenCart’s built-in mail system.
📊
Admin Dashboard
Full admin interface showing all active proposals, status history, and conversion rate from proposal to order.
🔐
Role-based Access
Separate permissions for proposal viewing, responding, and management. Integrates with OpenCart user groups.
🛒
One-Click Acceptance
When accepted, system automatically creates an order at the negotiated price. Zero friction to purchase.
